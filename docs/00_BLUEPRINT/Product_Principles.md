# ORION Product Principles

## Purpose

These principles guide every engineering, design, and product decision within ORION.

---

## Core Principles

### 1. One Platform, One Experience
Every module should feel like part of a single, unified product.

### 2. Every Screen Has One Purpose
Each screen should answer one important question for the user.

### 3. Information Before Decoration
Design should communicate clearly before it impresses visually.

### 4. Consistency Over Complexity
Reuse components, layouts, and patterns wherever possible.

### 5. Build for Long-Term Maintainability
Choose solutions that remain understandable and maintainable as ORION grows.

### 6. AI Should Assist, Not Overwhelm
Artificial Intelligence should help people make better decisions—not replace them.

### 7. Every Feature Must Improve Decision-Making or Efficiency
If a feature does neither, reconsider whether it belongs.

### 8. Design for Founders First
The Command Center should allow a founder to understand the platform in under 30 seconds.

### 9. Performance Is a Feature
Fast software is better software.

### 10. Simplicity Is a Competitive Advantage
The simplest solution that solves the problem is usually the best one.
