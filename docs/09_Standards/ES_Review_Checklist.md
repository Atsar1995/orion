# Engineering Specification Review Checklist

Before implementation, verify:

- [ ] Business objective is clear.
- [ ] Scope is defined.
- [ ] Out of scope items are listed.
- [ ] Deliverables are measurable.
- [ ] Acceptance criteria are testable.
- [ ] Definition of Done exists.
- [ ] Risks are documented.
- [ ] Dependencies are identified.
- [ ] Folder structure is specified.
- [ ] Existing functionality will not break.
- [ ] Documentation requirements are included.
- [ ] Testing requirements are included.

Only after every item is satisfied should implementation begin.
