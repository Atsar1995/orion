# ORION Engineering Standards

This directory contains the official engineering standards used throughout ORION.

These documents are considered frozen standards.

Changes require review and approval before adoption.

## Engineering Workflow

Every Engineering Specification must be reviewed and approved before implementation begins.

Every completed sprint must conclude with a Git commit documenting the work completed.

Engineering work should follow the standard ORION lifecycle:

Engineering Specification → Implementation → Architecture Review → Git Commit → Release Record
